FoxPRO-HR Step-by-Step Development Guide

Phase 1: Project Setup

  1. Create virtual environment and install requirements.
  2. Configure Django settings, static/media, authentication model.
  3. Create modular apps for each HR domain.

Phase 2: Data Model Layer

  1. Define all entities in app-specific models.py.
  2. Create relationships for HR workflows (employee to attendance/leave/payroll).
  3. Run makemigrations and migrate.

Phase 3: Admin + APIs

  1. Register models in each admin.py.
  2. Build CRUD views and REST endpoints per module.
  3. Add server-side validation and permissions.

Phase 4: Frontend

  1. Use templates/base.html for shared layout/navigation.
  2. Build dashboards and module pages under templates/dashboard/.
  3. Apply modern UI styling in static/css/styles.css.

Phase 5: Core Business Logic

  • Attendance late/overtime calculators
  • Leave balance and approval flow
  • Payroll engine and payslip PDF service
  • Recruitment pipeline and interview tracker

Phase 6: Deployment on PythonAnywhere

  1. Create PythonAnywhere web app (manual Django config).
  2. Upload project, set virtualenv, install requirements.
  3. Configure WSGI to point to foxpro_hr.wsgi.
  4. Set static files mapping and run migrations.
  5. Create admin user and reload web app.

See README and deployment guide for exact commands and file locations.
